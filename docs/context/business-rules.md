# Business Rules

This document consolidates all business rules for the KASIRSOLO platform. It serves as the single source of truth for business logic decisions.

---

## 1. Pricing Rules

### 1.1 Offline Plan (One-Time)
- One-time payment per app
- Price varies per app (Rp200k - Rp500k)
- License never expires after activation
- Includes 12 months of free updates
- After 12 months: app continues to work, no new feature updates without renewal
- No cloud sync, no portfolio site, no admin dashboard

### 1.2 Cloud Monthly Plan
- Monthly subscription per app
- Price equals the offline price per month
- All offline features + cloud sync + portfolio site + admin dashboard
- Can cancel anytime
- 7-day grace period after missed payment
- 30 days after expiry: license suspended

### 1.3 Cloud Yearly Plan
- Yearly subscription per app
- 20% discount compared to 12x monthly price
- Same features as cloud monthly
- Billed annually
- 7-day grace period after missed payment
- 30 days after expiry: license suspended

## 2. License Rules

### 2.1 License Key
- Format: `XXX-XXXX-XXXX-XXXX-XXXX` (3 letter prefix + 16 hex chars)
- Prefix is first 3 letters of app slug (uppercase)
- Generated by `ksp_generate_license_key()` function
- Unique across all licenses

### 2.2 License Status Transitions
```
trial --> active     (payment confirmed)
trial --> expired    (trial_ends_at passed)
active --> expired   (subscription lapsed + grace period)
expired --> active   (payment made / renewal)
expired --> suspended (30 days no payment)
suspended --> active (payment made + admin approval)
```

### 2.3 Trial Rules
- Duration: 14 days (configurable via ksp_settings)
- Max transactions during trial: 100
- Max products during trial: 50
- All features available during trial
- Plan type during trial: `offline` (no cloud sync)
- One trial per email address
- Trial does not require payment information

### 2.4 Offline License Rules
- Never expires after activation (`expires_at` = NULL)
- No cloud sync
- Data stored locally only
- 2 devices (independent data on each)
- 12 months of updates included
- License key validated once, then cached locally
- Periodic online check every 7 days (soft requirement)

### 2.5 Cloud License Rules
- Expires based on subscription period
- Full cloud sync between devices
- Data stored locally + cloud
- 2 devices (synced data)
- Continuous updates while subscription active
- License validated on every app startup (online required)
- Grace period: 7 days past due
- Suspension: 30 days past due

## 3. Device Rules

### 3.1 Device Limit
- Default: 2 active devices per license
- Configurable by admin: 1-10 devices (`ksp_licenses.max_devices`)
- Enforced at database level (trigger) and application level

### 3.2 Device Binding
- First access from a new device triggers binding
- Device identified by browser fingerprint
- Device assigned next available slot number (1, 2, ...)
- If max devices reached: binding blocked with error message
- User must unbind an existing device first

### 3.3 Device Unbinding
- Owner can unbind any device (remote unbind)
- Any user can unbind their own device (self-unbind)
- Unbinding sets `is_active = false`
- Unbinding does not delete device record (audit trail)
- Unbound device must re-bind to access the app

### 3.4 Device Heartbeat
- Active devices ping every 5 minutes
- Updates `last_seen_at` in ksp_devices
- Helps identify stale/unused devices

## 4. User & Role Rules

### 4.1 Roles
- **owner**: Full access. Created during registration. One per license.
- **manager**: Operational access. Can manage products, view reports. Cannot manage users or settings.
- **cashier**: POS access only. Can make sales. Cannot view reports or manage anything.

### 4.2 User Creation
- Owner creates other users (manager, cashier)
- Managers and cashiers belong to a specific license
- Email is required for all users but only owner/manager need Supabase auth
- Cashiers can use PIN-only access (6-digit PIN)
- PIN must be unique within a license

### 4.3 User Limits
- No explicit limit on number of users per license
- Only 2 devices can be active simultaneously
- Multiple users can share a device (switch via PIN)

## 5. Trial Rules (Detailed)

### 5.1 Trial Registration
- Requires: name, business_name, email, whatsapp, city, selected app
- Auto-generates password (sent via email)
- Creates: auth.user + ksp_client + ksp_license + ksp_user
- Trial starts immediately

### 5.2 Trial Period
- 14 calendar days from registration
- Full feature access (same as paid offline)
- Transaction limit: 100 (prevents using trial as permanent free tier)
- Product limit: 50

### 5.3 Trial Expiry
- Day 11: Warning notification (3 days remaining)
- Day 13: Final warning (1 day remaining)
- Day 14: Expired
  - App enters read-only mode
  - Can view existing data
  - Cannot create new transactions
  - Prominent "Aktivasi" CTA

### 5.4 Trial to Paid Conversion
- All trial data preserved when activating
- No data migration needed for offline plan
- Data migration needed for cloud plan (upload local to cloud)

## 6. Portfolio Site Rules

### 6.1 Eligibility
- Cloud plan only (monthly or yearly)
- One site per license (can be extended by admin)
- Included in cloud subscription (no extra cost)

### 6.2 Site Limits
- Max pages: 10 (configurable via ksp_settings)
- Max file upload size: 5MB per image
- Slug format: lowercase, alphanumeric, hyphens
- Custom domain: one per site
- No custom JavaScript (security)
- Limited custom CSS

### 6.3 Templates
- 4 templates: minimal, business, portfolio, landing
- Template can be changed anytime
- Content blocks are template-independent
- Template determines layout/style, not content

### 6.4 Publishing
- Sites start as unpublished (draft)
- Owner publishes when ready
- Published sites are publicly accessible
- Can be unpublished anytime

## 7. Payment Rules

### 7.1 Payment Methods
- Bank transfer (BCA, Mandiri, BNI, BRI)
- QRIS
- E-Wallet (OVO, GoPay, Dana)
- Manual verification by admin (current)
- Payment gateway integration (future: Midtrans/Xendit)

### 7.2 Invoice
- Format: `KSP-YYYYMMDD-NNNN`
- Generated automatically
- Amounts in IDR (BIGINT, no decimals)
- Tax: 0% currently (configurable)

### 7.3 Confirmation
- Client makes payment and confirms via WhatsApp
- Admin verifies payment in banking app
- Admin confirms in admin dashboard
- License activated automatically on confirmation

### 7.4 Refund
- Refund possible within 7 days of payment
- Pro-rata refund for subscription cancellation
- Refund creates a new transaction with status `refunded`
- License reverts to previous state

## 9. Data Rules

### 9.1 Prefix Convention
- `ksp_` = Cloud tables (Supabase PostgreSQL)
- `pos_` = Local stores (IndexedDB in browser)

### 9.2 Data Ownership
- All data belongs to the client
- Client can export data at any time
- Deleting a license does not auto-delete local data
- Cloud data archived for 90 days after subscription ends
- GDPR-style deletion available on request

### 9.3 Sync Rules (Cloud Plan)
- Write local first, sync to cloud asynchronously
- Server timestamp wins in conflicts
- Delta sync (only changed records)
- Sync queue processed in FIFO order
- Failed syncs retried with exponential backoff (max 5 attempts)
